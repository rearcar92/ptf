Project 01

네이버페이 연동

서비스에 외부 결제수단을 붙일 때 필요한 정책과 예외를 정리했습니다

회원, 결제, 운영 흐름에서 확인해야 할 정책과 예외 케이스를 정리하고 유관 부서와 적용 범위를 조율한 경험입니다.

Problem외부 결제수단 도입 시 확인 기준이 필요했습니다.
Approach연동 요건, 화면 흐름, 운영 처리 기준을 정리했습니다.
Role요구사항, 정책 검토, 협의 과정에 참여했습니다.
Project 02

카카오싱크 연동

가입·동의·회원정보 처리 흐름을 서비스 정책과 연결

가입과 동의 과정에서 사용자 경험과 데이터 처리 기준을 함께 봤습니다

동의 항목, 회원정보 매핑, 화면 문구, 예외 흐름을 기준으로 요구사항을 정리하고 적용 기준을 명확히 한 경험입니다.

Problem가입/동의 과정의 정보 기준이 필요했습니다.
Approach동의 항목과 회원정보 처리 기준을 정리했습니다.
Role정책, 화면 문구, 예외 흐름 검토에 참여했습니다.

다음 역할에서의 기여

커머스 연동, 정산, 운영 UX가 만나는 지점에서 복잡한 조건을 구조화하고 실행 가능한 기획 산출물로 정리하겠습니다.

추가 확장 가능 영역

ISMS 대응, PG 정산 업무, 관리자 화면 개선 사례는 지원 회사의 JD에 맞춰 별도 프로젝트 페이지로 확장할 수 있습니다.
